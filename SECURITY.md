# Security Policy — OpenFlow Governance

**HolyAI | holyai.dev**

---

## Reporting a Vulnerability

If you discover a vulnerability in the OpenFlow framework — logical, ethical, or technical — **do not open a public issue.**

Report privately to:  
📧 **holyai.dev** (contact form) or direct message **@pasha13** on GitHub

We will respond within **72 hours**.

---

## What Counts as a Vulnerability

- A loophole that allows bypassing the Zero-Authority axiom
- A flaw in the 6-Lens Podium that allows false certification
- A schema weakness enabling manipulated session logs
- A licensing gap enabling unauthorized commercial exploitation

---

## What We Commit To

- Acknowledge your report within 72 hours
- Investigate and respond with a fix timeline
- Credit you in the changelog (unless you request anonymity)
- Never retaliate against good-faith disclosures

---

## Scope

This policy covers the `holyai-dev/openflow-governance` repository only.
